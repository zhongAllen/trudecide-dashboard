## 1. 概述

**M2 驾驶舱**是 Trudecide 系统的首页（路由 `/dashboard`），定位为“CIO 视角的核心指标快速感知仪表盘”。核心理念是让用户在 5 秒内看清市场和个人投资组合的当前状态，以便快速做出是否需要深入分析的决策。

本次需求旨在完成驾驶舱的 DEMO 版本开发，实现核心功能布局和数据展示，为后续的自动化和 AI 增强功能奠定基础。

## 2. 整体设计

- **布局风格**: 遵循现有系统的浅色主题（白底蓝绿色调），采用卡片式布局，确保页面干净、简洁、重点突出。
- **数据来源**: DEMO 阶段，所有数据优先读取数据库真实数据。若无真实数据，则使用 Mock 数据并明确标注 `⚠ 测试数据`。
- **交互**: 支持页面内编辑模式，允许用户自定义关注的指标看板。

## 3. 页面布局与功能模块

驾驶舱页面由上到下分为 5 个核心区域：

### 3.1 区域 1：顶部摘要栏

此区域为全宽单行布局，横向排列 4 个核心数字卡片，提供最高层级的状态概览。

| 卡片名称 | 数据来源 | 备注 |
|---|---|---|
| **总资产** | `portfolio_holdings` 表（手动维护） | 显示账户总资产净值。 |
| **今日盈亏** | `portfolio_holdings` 表 + `stock_daily` 表 | 实时计算当日盈亏金额及百分比，红绿颜色区分。 |
| **累计收益率** | `portfolio_holdings` 表 | 对比沪深300指数的累计收益率。 |
| **宏观信号灯** | 宏观矩阵综合评估结果 | 显示“强多头/弱多头/中性/空头”之一。 |

### 3.2 区域 2：当前持仓表现

此区域位于页面左侧（约占 40% 宽度），以表格形式展示当前持仓的详细情况。

- **数据来源**: `portfolio_holdings` 表。
- **表格字段**: 股票名称/代码、持仓成本、现价、盈亏金额、盈亏百分比、持仓逻辑摘要（一句话）。
- **交互**: 表格底部提供“管理持仓”按钮，点击后可跳转至持仓录入/管理页面。

### 3.3 区域 3：可配置指标看板

此区域位于页面右侧（约占 60% 宽度），采用 2xN 网格布局，允许用户自定义关注的核心宏观指标。

- **默认指标**: 默认展示 6 个关键宏观指标，包括制造业PMI、CPI同比、M2同比、LPR、10年期国债收益率、北向资金月度净流入。
- **数据来源**: `indicator_meta` 及 `indicator_values` 表。
- **交互**: 
    - 每个卡片包含指标名称、最新值、环比变化及迷你折线图。
    - 右上角提供“编辑”按钮，进入编辑模式后，用户可从 `indicator_meta` 表中勾选/取消任意指标，实现高度自定义。
- **配置存储**: 用户配置将保存至 `dashboard_config` 表，预留 `user_id` 字段为后续多用户独立配置做准备。

### 3.4 区域 4：双地图可视化

此区域并排展示两个地图，提供宏观和市场的地理空间视角。

- **左：全球市场气泡地图**
    - **功能**: 在世界地图上，通过气泡大小和颜色展示全球主要股票市场的今日涨跌幅。
    - **覆盖市场**: A股（沪深300）、美股（标普500/纳斯达克）、港股（恒生指数）、日经225、欧洲主要指数（富时100/德国DAX）。
    - **数据来源**: `index_daily` 表。

- **右：中国省份热力图**
    - **功能**: 根据 A 股上市公司注册地所属省份，以颜色深浅展示各省上市公司的今日平均涨跌幅。
    - **数据来源**: `stock_daily` 表与 `stock_meta.area` 字段。
    - **交互**: 点击任一省份，可弹出该省今日涨跌幅 Top 5 的股票列表。

### 3.5 区域 5：新闻大事件快览

此区域位于页面底部，全宽布局，旨在提供今日市场关键信息摘要。

- **数据来源**: `cctv_news` 表（新闻联播）及 AI Agent（未来）。`news_raw` 表当前为空，暂不使用。
- **布局**: 
    - **左侧**: “今日重要新闻列表”，展示来自 `cctv_news` 的新闻标题、来源和时间。
    - **右侧**: “AI 每日摘要”卡片，DEMO 阶段使用 Mock 数据展示“今日最重要的3件事”，为后续 AI Agent 生成摘要预留位置。

## 4. 数据库变更

为支持驾驶舱功能，需新增以下两张表：

### 4.1 `portfolio_holdings` 表

用于记录用户持仓情况。

| 字段名 | 类型 | 描述 | 备注 |
|---|---|---|---|
| `id` | `uuid` | 主键 | `default gen_random_uuid()` |
| `user_id` | `uuid` | 用户ID | 预留字段，关联 `auth.users` |
| `ts_code` | `text` | Tushare 股票代码 | |
| `name` | `text` | 股票名称 | |
| `cost_price` | `float8` | 持仓成本价 | |
| `shares` | `int8` | 持仓数量 | |
| `strategy_note` | `text` | 持仓逻辑摘要 | 一句话描述为何买入 |
| `source` | `text` | 数据来源 | `manual` 或 `qmt` |
| `created_at` | `timestamptz` | 创建时间 | `default now()` |
| `updated_at` | `timestamptz` | 更新时间 | `default now()` |

### 4.2 `dashboard_config` 表

用于存储用户自定义的指标看板配置。

| 字段名 | 类型 | 描述 | 备注 |
|---|---|---|---|
| `id` | `uuid` | 主键 | `default gen_random_uuid()` |
| `user_id` | `uuid` | 用户ID | 预留字段，关联 `auth.users` |
| `indicator_ids` | `jsonb` | 指标ID数组 | 存储用户选择的 `indicator_meta.id` 列表 |
| `updated_at` | `timestamptz` | 更新时间 | `default now()` |
