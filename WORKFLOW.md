# 开发流程规范

## 每次代码变更后的标准流程

### 1. 本地检查（必须）
```bash
# TypeScript 类型检查
npm run check

# 本地构建测试
npm run build

# 确认变更文件
git diff --stat
```

### 2. 提交前检查清单
- [ ] 类型检查无错误（或确认错误非本次引入）
- [ ] 构建成功
- [ ] 变更文件符合预期
- [ ] 版本号已更新（如需要）

### 3. 提交并推送
```bash
git add -A
git commit -m "版本: 描述

- 变更点1
- 变更点2

🤖 Generated with [Qoder][https://qoder.com]"
git push
```

### 4. 部署后验证（5分钟后）
- 检查 Vercel 部署状态
- 验证线上功能正常

---

## 错误处理流程

### 如果类型检查有错误
1. 修复错误
2. 重新运行 `npm run check`
3. 确认构建成功后再推送

### 如果构建失败
1. 查看错误日志
2. 修复问题
3. 重新构建直到成功

### 如果部署失败
1. 查看 Vercel 部署日志
2. 修复问题
3. 重新推送
